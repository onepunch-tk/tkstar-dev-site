## Goal

T021 통합 QA 중 발견된 production 배포 blocker 회귀를 해결한다. `app/presentation/components/content/MdxRenderer.tsx` 의 `evaluateMdxBody` 가 `new Function(code)({Fragment, jsx, jsxs})` 로 velite `s.mdx()` 함수-바디를 runtime 평가하는데, T007 (PR #14, 2026-04-27) 시점에는 Cloudflare Workers V8 isolate 가 이를 허용했으나 그 이후 정책 강화로 default disallow 가 되어 4개 detail 라우트가 500 (`EvalError: Code generation from strings disallowed for this context`).

## Context

- **Why**: detail 페이지 4종 (`/projects/:slug`, `/blog/:slug`, `/legal/:app/{terms,privacy}`) 이 production 환경에서 500 응답 → T022 배포 차단.
- **회귀 원인**: T007 PR #14 의 `wrangler.jsonc` NOTE ("Cloudflare Workers V8 Isolate currently permits this without `unsafe-eval`") 가 그 이후 무효화. wrangler 4.85 + compatibility_date 2026-04-27 환경에서 차단 확인.
- **관련 PRD 섹션**: PRD `F005 Project Detail` / `F007 Blog Detail` / `F014 App Legal Doc`
- **관련 PROJECT-STRUCTURE 디렉토리**: `app/presentation/{components/content, routes}`, `app/{domain,infrastructure}/content/{**, ../mappers}`, `velite.config.ts`, `vite.config.ts`

## 채택 전략 — `@mdx-js/rollup` + Vite + `import.meta.glob`

> **Issue #75 본문이 명시한 mdx-bundler 도입은 채택하지 않는다.**

### 근거 (mdx-bundler 폐기)

`mdx-bundler/src/client/react.js` 의 `getMDXExport` 직접 인용:

```js
function getMDXExport(code, globals) {
  const jsxGlobals = {React, ReactDOM, _jsx_runtime}
  const scope = {...jsxGlobals, ...globals}
  // eslint-disable-next-line
  const fn = new Function(...Object.keys(scope), code)
  return fn(...Object.values(scope))
}
```

→ 우리 현재 `evaluateMdxBody` (`new Function(code)({Fragment, jsx, jsxs})`) 와 본질적으로 동일한 패턴. V8 isolate `Code generation from strings disallowed` 차단을 100% 그대로 받는다. Issue #75 본문 표현 ("Cloudflare Workers SSR 공식 권장 패턴") 은 부정확.

### 근거 (`@mdx-js/rollup` 채택)

- MDX 공식 docs (mdxjs.com) 1순위 Vite 통합 패턴.
- Vite 빌드 시점에 `.mdx` → 진짜 ESM 모듈로 변환 → runtime eval 0건 → V8 차단 회피.
- `@vitejs/plugin-react` 와 `enforce: 'pre'` 로 함께 사용.
- `import.meta.glob({ eager: true })` 로 4개 콘텐츠 모듈을 SSR 환경에서 sync lookup → React Router loader 의 직렬화 제약 회피.
- 이미 우리 스택 (`@cloudflare/vite-plugin` 1.33.2 + React Router 7.14.0 + Vite 8.0.3) 과 자연 통합 — 추가 장치 최소.
